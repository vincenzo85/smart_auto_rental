# User Stories

## Lato Utente (Customer)
1. Come cliente voglio registrarmi/login per poter prenotare auto in autonomia.
2. Come cliente voglio cercare auto disponibili per sede, date e categoria per trovare l'opzione migliore.
3. Come cliente voglio vedere il prezzo stimato con dettagli (weekend, sconto, assicurazione) per capire il costo totale.
4. Come cliente voglio prenotare un'auto e pagare online per ricevere conferma immediata.
5. Come cliente voglio scegliere "pay at desk" per concludere in filiale.
6. Come cliente voglio annullare una prenotazione e vedere rimborso/penale.
7. Come cliente voglio entrare in waitlist se non c'e disponibilita.
8. Come cliente voglio consultare lo storico/audit eventi della mia prenotazione.

## Lato Amministratore/Operatore
1. Come admin voglio creare/modificare/disabilitare auto per gestire la flotta.
2. Come operatore voglio pianificare manutenzioni per bloccare disponibilita in modo corretto.
3. Come admin voglio visualizzare report auto piu noleggiate per decisioni su flotta.
4. Come admin voglio monitorare utilization per sede per ottimizzare asset.
5. Come admin voglio ricevere audit trail per troubleshooting e compliance.
6. Come operatore voglio ritentare pagamenti falliti/pending quando necessario.

## Lato Programmatore/Team
1. Come sviluppatore voglio una struttura modulare chiara per mantenere ed estendere rapidamente il codice.
2. Come sviluppatore voglio migration versionate per garantire coerenza schema DB.
3. Come sviluppatore voglio test unit/integration con Testcontainers per ridurre regressioni.
4. Come DevOps voglio docker-compose per avviare stack locale in modo ripetibile.
5. Come reviewer voglio regole chiare di naming, DTO boundary e error handling standard.
6. Come PM voglio roadmap v2/v3 e changelog per pianificare rilascio incrementale.
